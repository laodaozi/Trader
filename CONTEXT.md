## 用户与场景

- **用户**：Scott（个人交易决策者）
- **场景**：每天开盘前（和日报同步更新），拿到一份对自选池 40 只股票的诊断和交易策略，辅助当日操作决策
- **现状**：手工翻 K 线判断买卖点 → 替代为系统自动输出"今天看哪些、怎么操作"
- **池子来源**：`/Users/scott/Desktop/一小步/stock panel.xlsx`（40 只：A 股 37 + ETF 512480/512880 + 美股 XNET + 港股 2350）
- **时间**：盘前输出，和日报 `trader.py` 运行时间同步

## 成功标准

- [ ] 开盘前可查看一份自选池策略页面（40 只全出，按优先级排序）
- [ ] 前 5 标 ⭐，只对有信号的股票输出完整买卖建议
- [ ] 每只输出结构化数据并存 JSON 日志（代码/日期/信号/建议），供日后回测反思
- [ ] 输出稳定：每日可复现，不因单只 API 失败全页面崩溃

## 范围

**做**：
- 读取 Excel 自选池 → 逐只诊断 → NX 信号解读 → scanner 模型命中叠加 → 策略分类（进攻/买入/埋伏 + 波段/趋势）→ 优先级排序 → HTML 页面 + JSON 日志
- 信号类型：进攻信号 / 买入 / 埋伏
- 交易风格：波段（NX 逻辑为核心）/ 趋势
- 入场/止损/目标位建议
- 结构化日志：代码 + 日期 + 信号 + 建议 + 后续可追加实际结果

**不做**：
- 自动下单 / 接入券商 API / 实时盯盘推送
- 多策略组合优化（日后做）
- 反思训练进化引擎（日后做，现阶段只记录结构化日志）

## 诊断链设计（已确认）

```
signals.py（NX 买点 + MA 排列 + Fib 位）
    ×
diagnose.py（周线方向 + 主力资金 + R:R）
     ×
scanner overlay（14 模型命中作为共振加分项，含好运低吸/牛市第一阳/超强反弹）
     ↓
strategy.py → 策略分类 + 优先级排序 → HTML 输出 + JSON 日志
```

理由：`diagnose` 覆盖周线方向+资金+R:R，`signals` 覆盖日线买卖点，二者结合是"诊断→策略"最短路径。scanner 模型命中叠加作为置信度加分（如 "NX 买点 + htji 回调狙击同时命中 = 高置信度"）。

- ⚠️ 信号过期逻辑（+3 交易日）需应用到策略页面

## 术语表

| 术语 | 定义 |
|------|------|
| NX 信号 | 典型价 PRINT = (3C+H+L+O)/6 的 RSI(6) 转折点，buy/sell/rising/neutral |
| 优选 | 40 只全出，按优先级排序，前 5 标 ⭐ |
| 进攻信号 | NX buy + MA 多头 + 资金净流入 → 高置信度买入 |
| 买入 | NX buy 或模型命中，中等置信度 |
| 埋伏 | Fib 支撑位附近 + R:R ≥ 2，低吸等待 |
| 波段 | 短期操作（持有 3-10 天），NX 逻辑为核心 |
| 趋势 | 中期操作（持有 10-30 天），MA 多头排列为背景 |
